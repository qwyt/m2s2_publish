How to use

Quick start guide Simple as 1, 2, 3

  1. Use each of the tabs at the top to specify your HTML page's content, libraries, metadata and resources. All changes are applied immediately to your generated template.
  2. View your generated template under the Result tab.
  3. When satisfied with the results, copy it to your clipboard or export it to JSFiddle or Codepen using the buttons at the very top of the Result tab.

Extended guide For the demanding user

Content tab
  • The Content tab allows you to specify your page's content and customize your template. You can define a title for your page's <head>, classes for the <body>, as well as your own customized content for the <body> element. Your user-defined content can be placed before or after the generated template.
  • We offer a variety of simple templates, ranging from simple empty pages, showcases and clean layouts to basic form templates and error pages for all your needs. Most template bases can be customized in the second section of the Content tab.
  • Finally, you can enable the automated comments, so that you know what is what in the final results.

Libraries
  • The Libraries tab offers a wide variety of popular Javascript and CSS libraries from which to choose for your template. By default, your template will have jQuery 3.1.0 included, along with a minimal boilerplate code snippet in the page's <head> to help get you started.
  • You can enable as few or as many libraries as you like. The counter at the top of the Libraries tab will keep track of the amount of loaded libraries. Be careful not to add too many libraries, otherwise your page might be slow.
  • Dependencies will be resolved automatically, although you will be notified which libraries will be included based on your selection. If two or more libraries are mutually exclusive (i.e. different versions of the same library), they last one selected will automatically cause the others to deselect.
  • Some libraries allow you to include a snippet of boilerplate code in the page's <head> to help get you started. You can choose if you want this enabled for each library you include, using the second section of the Libraries tab.
  • Some other libraries will offer different options, such as the different versions of Bootstrap, which allow you to style your generated template using some of their styles.

Metadata
  • The Metadata tab allows you to specify your page's metadata tags, such as character encoding, author, description, keywords, generator, as well as some other properties for your page's <head>, such as viewport, favicon and warning for older browsers.
  • Some options are selected by default (such the inclusion of generator and viewport tags), but you can turn them off if you do not want them.
  • The different versions of Bootstrap require the viewport tag to be included to work properly, so remember to toggle it on if you include any of them.

Resources
  • The Resources tab lets you specify your page's external Javascript and CSS resources, using either relative or absolute paths.
  • For each of the two types of resource, use the corresponding textarea and specify one resource's location per line.
  • If any commas or spaces are found in your input, you will be notified, however the generated template will include those resources, even if they do not work properly.

Result
  • The Result tab allows you to view your generated page. It updates instantly after you make any changes in the other tabs to reflect them. The code itself, however, cannot be edited.
  • You can use the buttons at the very top of the panel to copy the resulting template page to your clipboard or export the template page to JSFiddle or Codepen.
  • If you export to JSFiddle or Codepen, some changes like resources with relative paths might be omitted.

Miscellaneous
  • Clicking the Reset button will present you with a dialog, asking if you are sure you want to reset the generator. All changes will be lost and you will have to start over.
  • The Help & Info tab shows you the page's quick start and extended guides, as well as the About section, which includes information about the project itself. If you find this tool useful, please Star it on Github.
  • For any errors encountered or feature requests, use the Issue button in the About section of the Help & Info tab.
